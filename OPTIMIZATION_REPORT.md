## 1️⃣ immortalwrt.config

### 必要性：★★★★★ (必需)

**作用**：OpenWrt 编译配置文件，决定固件包含哪些功能和包。

### 优化内容：

#### ✅ 已改进
1. **结构化分类** - 按功能模块分组（核心网络、DNS、代理工具、加密库等）
2. **添加详细注释** - 每个模块标注"必需/推荐/可选"
3. **移除冗余配置** - 删除重复的 IPv6 配置项
4. **清理无用选项** - 将不需要的包统一移至"已禁用"区域
5. **移除过时版本** - 删除 `CONFIG_NODEJS_16`（已有 node108）

#### ❌ 已移除的不必要项
| 配置项 | 原因 |
|--------|------|
| `CONFIG_LIBCURL_*` (大部分) | libcurl 默认已包含基本功能 |
| `CONFIG_MBEDTLS_DES_C` | DES 已淘汰，不需要 |
| `CONFIG_MBEDTLS_ECP_DP_SECP256K1_ENABLED` | 仅比特币用，一般不需要 |
| `CONFIG_PCRE*_JIT_ENABLED` | JIT 可能兼容性问题 |
| `CONFIG_PACKAGE_libdb47` | 过时版本，易有安全漏洞 |
| `CONFIG_PACKAGE_kmod-inet-diag` | 诊断工具，非必需 |
| `CONFIG_PACKAGE_ruby-bigdecimal` 等 | Ruby 子模块按需加载 |

#### 💡 建议
- 如不需多个代理工具，可只保留一个（推荐 homeproxy 或 openclash）
- SNMP 监控如不用可移除，节省约 500KB

---

## 2️⃣ zzz-default-settings

### 必要性：★★★★☆ (高)

**作用**：系统首次启动时的初始化配置脚本。

### 优化内容：

#### ✅ 已改进
1. **添加错误处理** - 所有命令添加 `2>/dev/null` 或 `|| true`
2. **简化版本信息写入** - 使用 heredoc 替代多次 sed
3. **移除危险操作** - 删除状态页文件删除操作
4. **优化无线启用** - 添加失败容忍
5. **注释 DNS 重定向** - 避免影响正常 DNS 解析
6. **循环禁用服务** - 代码更简洁

#### ⚠️ 原问题
| 问题 | 风险 | 修复方式 |
|------|------|----------|
| `rm -rf /usr/lib/lua/luci/view/admin_status/index` | 可能导致 LuCI 状态页崩溃 | 已移除 |
| 多个 sed 修改 version.lua | 容易出错且难维护 | 改用 heredoc |
| `wifi up` 无错误处理 | 无无线硬件时会报错 | 添加 `|| true` |
| DNS 重定向默认启用 | 可能导致 DNS 污染 | 改为注释 |

#### 💡 建议
- 如需修改 LAN IP，建议在 `customize.sh` 中完成
- 时区设置对国内用户是必需的

---

## 3️⃣ customize.sh

### 必要性：★★★☆☆ (中)

**作用**：在编译前修改 OpenWrt 源码的默认配置。

### 优化内容：

#### ✅ 已改进
1. **添加详细注释** - 每项功能标注必要性和说明
2. **添加错误处理** - `set -e` 确保失败时立即退出
3. **IP 地址转义** - `192\.168\.1\.1` 避免正则问题
4. **可选功能注释** - 密码设置等敏感操作默认关闭
5. **进度输出** - 添加执行状态提示

#### 📊 功能评估
| 功能 | 必要性 | 建议 |
|------|--------|------|
| 修改默认 IP | ★★★☆☆ | 保留，避免冲突 |
| 修改主机名 | ★★☆☆☆ | 可选 |
| 设置 root 密码 | ★☆☆☆☆ | 生产环境应手动设置 |
| 启用 upx 压缩 | ★★☆☆☆ | 需要额外编译时间 |

#### 💡 建议
- 默认 IP 修改建议保留（10.1.1.110 较冷门）
- root 密码建议首次登录后手动设置
- 如不需要特殊定制，此文件可简化为只有 IP 修改
